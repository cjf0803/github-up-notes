1.建立一个github库


2.配置ssh秘钥 这个操作就是让我们电脑和github进行一个绑定验证
	1.右键鼠标找到Git Bash Here 
	2.需要配置的ssh命令：ssh-keygen-t rsa -C "xxx@qq.com"
	3.注意一路回车就可以 不需要键入yes啥的
	4.在电脑C盘中找到对应的路径 并复制秘钥 到git SSH秘钥编辑处添加即可
        ps:测试命令：ssh -T git@github.com 如果成功则会显示 Hi,successfully ..............

3.配置用户密码：
	同样是在git上敲命令
		1.git config --global user.name "xxx"
		2.git config --global user.email "xxxxx"

4.上传代码：

	ps:1.建议给github上传 新建一个文件夹（方便以后代码上传）
	     2.在 github 上只能删除仓库，却无法删除文件夹或文件，所以只能通过 git 的基本命令来解决。
-----------------------------------------------------------
	1.初始化仓库(在新建的github 上传文件下 右键打开gitbash )
	     git init	

	2.首先进入你的 master/main 文件夹下。
	     git pull origin master/main
	3.开始添加文件
		1.git add 文件名（将“指定”内容从工作目录添加到暂存区）
		2.git add . (将文件夹下“所有”内容从工作目录添加到暂存区)
	4.开始提交(将暂存区文件提交到.git目录下)
		git commit -m "xxxx(描述)"
	5.通过库的ssh地址进行远程连接（可以放在最开始敲这个命令：先连接库）
		git remote add origin （将本地库的ssh地址复制下来）

	6.提交到库中（将git目录下文件提交到远程仓库中）
		git push -u origin master/main
		**ps:此时可能会出现问题 当库中有readme.md文件时需要先把库中文件下载下来再提交
			命令为：git pull --rebase origin main
				然后再执行git push -u origin master/main 即可上传成功

5.删除 库中的某个文件
	1.同样 先进入到你的 master/main 文件夹下。
		git pull origin master/main
	2.
	step1: git rm 文件 //本地中该文件会被删除
	step2: git rm -r 文件夹 //删除文件夹
	step3: git commit -m '删除某个文件'
	step4: git push -u（origin master/main）
	3.
	上面的方法会把对应的本地文件也删除掉，如果不想把本地文件删除，只把缓存区中的对应部分删除		，则加上--cached
	git rm --cached 文件 //本地中该文件不会被删除
	git rm -r  --cached  文件夹 //删除文件夹
